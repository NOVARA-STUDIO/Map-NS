# Фото-мапа

Сайт-«мапа», зібрана з фотографій, розміщених одна біля одної (як тайли Google Maps). Можна перетягувати мишею та наближати/віддаляти колесом миші. Краї фото плавно перетікають одне в одне (прозорість на стиках).

## Демо-структура

- `index.html` — сторінка
- `style.css` — стилі
- `config.js` — **тут налаштовується мапа**: розмір фото, точні координати кожного тайлу, прозорість на стиках
- `app.js` — логіка побудови сітки, перетягування та зуму
- `align_photos.py` — скрипт, який сам обчислює точні координати з ваших фото (не треба підбирати на око)
- `photos/` — сюди кладете свої фото

## Чому важливі точні координати, а не просто "рядок × стовпець"

Реальні фото рідко накладаються одне на одне строго по прямій сітці — буває невеликий діагональний зсув (наприклад, сусіднє знизу фото ще й трохи зсунуте вбік). Тому кожен тайл у `config.js` має власні `x`, `y` — точну позицію в пікселях, а не обчислену механічно за номером рядка/стовпця.

## Як додати свої фото (автоматично, рекомендовано)

1. Назвіть фото за схемою `рядок_стовпець.jpg` з нумерацією **з 0** (`0_0.jpg`, `0_1.jpg`, `1_0.jpg`, `1_1.jpg` …) і покладіть у папку `photos/`.
2. Встановіть залежності (один раз):
   ```bash
   pip install numpy scipy pillow --break-system-packages
   ```
3. Запустіть скрипт вирівнювання:
   ```bash
   python3 align_photos.py --dir photos --out config.js
   ```
   Він сам знайде, наскільки і в якому напрямку кожне фото накладається на сусідні (аналізуючи саме зображення, а не назви файлів), і згенерує готовий `config.js`.
4. Відкрийте `index.html` — мапа має зійтися без розривів.

Якщо якесь фото "не прив'язалося" — скрипт про це напише в консоль (значить, у нього немає спільного сусіда з іншими файлами).

## Як додати фото вручну (без скрипта)

У `config.js` для кожного тайлу вкажіть:
- `row`, `col` — логічна адреса (яке з яким сусідить, для прозорих країв),
- `x`, `y` — точна позиція в px відносно тайлу `(0,0)` (можна виміряти в будь-якому фоторедакторі, суміщаючи спільні деталі на фото),
- `src` — шлях до файлу.

```js
const MAP_CONFIG = {
  tileWidth: 1080,
  tileHeight: 1080,
  edgeFade: true,
  fadeWidth: null,
  tiles: [
    { row: 0, col: 0, x: 0,   y: 0,   src: "photos/0_0.jpg" },
    { row: 0, col: 1, x: 508, y: 0,   src: "photos/0_1.jpg" },
    { row: 1, col: 0, x: 38,  y: 432, src: "photos/1_0.jpg" },
    { row: 1, col: 1, x: 461, y: 432, src: "photos/1_1.jpg" },
  ],
};
```

`edgeFade: true` вмикає плавну прозорість на стиках (фейдиться вся зона фактичного накладення). `fadeWidth` можна задати числом (px), щоб обмежити ширину зони згасання, якщо накладення дуже велике.

## Як опублікувати на GitHub Pages

1. Створіть новий репозиторій на GitHub (наприклад `photo-map`).
2. Завантажте туди всі файли з цієї папки (включно з `photos/` та вашими фото).
3. У репозиторії відкрийте **Settings → Pages**.
4. У розділі **Build and deployment** оберіть джерело **Deploy from a branch**, гілку `main` і папку `/ (root)`.
5. Збережіть — за хвилину-дві сайт буде доступний за адресою на кшталт:
   `https://<ваш-нікнейм>.github.io/photo-map/`

Або через термінал:

```bash
git init
git add .
git commit -m "Фото-мапа"
git branch -M main
git remote add origin https://github.com/<ваш-нікнейм>/photo-map.git
git push -u origin main
```

Потім увімкніть GitHub Pages, як описано вище.

## Керування на сайті

- **Перетягування мишею / пальцем** — рух по мапі
- **Колесо миші / щипок двома пальцями** — зум
- Кнопки **+ / – / ⤢** внизу праворуч — зум і скидання вигляду до всієї мапи
